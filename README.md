# ozaretskyi / status-page

GitHub Actions status dashboard for **starred repositories**, auto-rebuilt every 15 minutes via GitHub Actions.

Shows only repos you've starred on GitHub (via `GET /user/starred`). To add or remove a repo from the dashboard, star or unstar it on GitHub — the next build picks it up automatically.

## Setup

### 1. Create PAT
Go to **GitHub → Settings → Developer settings → Personal access tokens → Fine-grained tokens**

Required permissions (repository access: **All repositories**):
- `Actions` — Read
- `Metadata` — Read

### 2. Add secret
Go to this repo → **Settings → Secrets and variables → Actions → New repository secret**

- Name: `STATUS_PAGE_TOKEN`
- Value: your PAT

### 3. Enable GitHub Pages
Go to **Settings → Pages**
- Source: **Deploy from a branch**
- Branch: `main` / `/(root)`

### 4. Trigger first build
Go to **Actions → Build & Deploy Status Page → Run workflow**

The page will be live at `https://<username>.github.io/<repo-name>/` (or your custom domain).

## Structure

```
├── index.template.html   ← edit this for customisation
├── index.html            ← auto-generated by CI (contains injected token)
├── .github/
│   └── workflows/
│       └── build.yml
└── .gitignore
```

## ⚠️ Security note
`index.html` is generated by CI and contains the PAT in plain text (client-side JS).
Keep this repo **private**. The PAT itself only needs read-only Actions + Metadata scopes,
so exposure risk is minimal, but private repo is the right call.
